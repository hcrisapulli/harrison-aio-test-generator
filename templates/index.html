Best Practices (click to expand)
  • Keys are auto-generated from your prefix — no need to fill them manually.
  • Description is optional — leave blank if the title is self-explanatory.
  • Pre-condition should only capture test-specific setup (e.g. "Entity connected to Xero"). Do not write "User is logged in" — that is implied.
  • Tags should be relevant to the test. Only add Regression if explicitly required.
  • Steps & Expected Results are paired line-by-line. Each line becomes a separate row in the CSV.
  • Status is always set to Draft automatically.
  • Keep test suites focused — test the main feature, not every edge case.
